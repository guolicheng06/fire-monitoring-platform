### 1.1 技术栈
- **后端**:PHP 语言，ThinkPHP框架

## 二、UI设计规范（深色主题）

### 2.1 设计目标
统一、专业、低视觉疲劳的深色主题，以黑色/深灰为主基调，全系统视觉一致。

### 2.2 色彩规范

| 用途 | 颜色值 |
|------|--------|
| 主背景色 | `#0A0A0A` |
| 次级背景 | `#1E1E1E`（卡片、侧边栏、表格） |
| 强调色 | `#0078D4`（按钮、链接、选中状态） |
| 主要文字 | `#E5E5E5` |
| 次要文字 | `#A0A0A0` |
| 禁用/占位文字 | `#6C6C6C` |
| 边框/分割线 | `#2D2D2D` |
| 错误/警告 | `#D32F2F`（可适当降低饱和度） |

### 2.3 组件与布局
- **头部导航栏**:高 56px，背景 `#0F0F0F`，底部 1px 分割线
- **侧边菜单**:宽 240px，背景 `#111111`，菜单项 hover 背景 `#2A2A2A`，选中项左侧加 3px 强调色竖条
- **卡片容器**:圆角 8px，背景 `#1E1E1E`，内边距 20px，阴影 `0 2px 8px rgba(0,0,0,0.3)`
- **按钮**:
  - 主按钮:实心强调色背景，白色文字，圆角 6px
  - 次按钮:透明背景 + 强调色边框
  - 危险按钮:使用红色系
- **表格**:
  - 表头背景 `#1A1A1A`，文字加粗
  - 行分隔线 `#2D2D2D`
  - hover 行背景 `#2A2A2A`
- **表单控件**:
  - 输入框背景 `#222222`，边框 `#3A3A3A`，聚焦时边框变为强调色
  - 下拉框、日期选择器等保持相同风格

### 2.4 图标与图片
- 图标采用线性风格（Feather Icons / Material Icons Outlined），颜色随文字色，hover 变为强调色
- Logo 使用白色版本适配深色背景

### 2.5 交互反馈
- 所有可点击元素提供 hover 效果
- 提示使用深色风格 Toast（如 `message` 组件），加载使用骨架屏或深色加载动画

### 2.6 统一性保障
- 建立全局 CSS 变量（`--bg-primary`, `--text-primary` 等）
- 所有页面基于同一套组件库开发，禁止硬编码颜色

## 三、权限管理设计

### 3.1 设计目标
实现基于角色（RBAC）的**菜单级**与**操作级**权限控制，支持“低敏感菜单默认可见”。

### 3.2 核心概念
- **用户**:可关联多个角色
- **角色**:权限集合载体，内置“超级管理员”不可删除
- **权限**:菜单可见性 + 操作（增/删/改/查）
- **低敏感菜单**:如“首页”、“任务列表（查看）”等，默认对任何登录用户可见，无需分配

### 3.3 功能需求

#### 3.3.1 角色管理
- 支持增、删、改角色
- 超级管理员拥有所有菜单和所有操作权限，角色标识 `is_super_admin = 1`

#### 3.3.2 权限分配（管理员操作）
- **菜单分配**:在角色配置界面，显示菜单树，低敏感菜单默认勾选且不可取消，其他菜单由管理员手动勾选
- **操作权限分配**:针对每个菜单，可分配具体操作（如“新增”、“编辑”、“删除”、“查看”）

#### 3.3.3 权限校验
- **菜单渲染**:登录后返回用户可见菜单列表（默认菜单 + 分配菜单），前端动态生成菜单
- **操作校验**:前端根据权限标识控制按钮显隐，后端对每个敏感接口进行权限校验

#### 3.3.4 权限优先级
- 多角色权限取**并集**
- 超级管理员忽略所有权限限制

## 六、功能详情（修正与补充）

### 6.1 首页
- **主页展示**:按中控大屏样式布局，包含六个模块
  1. **设备数据曲线图**:显示首页设置中选中的设备数据（device.curve/index?id=device_id,device_id从fa_mainconfig读取）。若无设置，显示提示并跳转设置页（mainconfig/index）
  2. **AI对话框**:显示最新10条设备报警消息，按时间倒序(device.alarm/index)（显示内容：设备名称，设备报警内容messagecontent）
  3. **BIM模型**:使用Three.js加载 glTF/glb 文件（不支持rvt，需预先转换）。若无模型，显示空状态,若没有设置，显示提示并跳转设置页（mainconfig/index）
  4. **设备类型统计**:按设备类型分组统计数量，最多展示3种（fa_device表的type字段）
  5. **设备状态统计**:按设备当前状态（正常/异常/维护中）统计数量，最多展示2种（fa_device表的current_statusdata字段）
  6. **历史记录**:显示最近5条设备维护记录（按时间倒序）(fa_device_maintain，参考device.maintain/index)(显示数据：设备名称，状态statusdata，维护内容construction_recordcontent)
  **6个模块布局说明**:首页菜单是系统监测的核心，所以要求该界面可以参考中控大屏设计。整个界面分为水平分3个部分（按比例左：中：右=1：4：1）:左侧（2个模块1:1比例上下显示，模块1-设备数据曲线图和模块2-AI对话框）；中间分为上下比例为4:1（上面是显示模块3-BIM模型三维图，下面是分为左右两个部分，左边是模块4-设备类型右边是模块5-设备状态）；右侧（1个模块，模块6-历史记录）
- **首页设置**:
  - 选择曲线图设备（从设备列表选择）
  - 上传BIM模型（仅支持glTF/glb，大小限制50MB）

### 6.2 商户管理
- **商户列表**:
  - 筛选:业态类型
  - 操作:添加、批量删除
  - 列表字段:ID、名称、业态、面积、层高、夜间营业、评分、操作（详情、编辑、删除）
  - 点击行进入商户详情

- **商户详情**:
  - 上半部:商户基本信息
  - 左下:雷达图（火灾荷载、点火源、人员密度、疏散难度、探测器分数）
  - 右下:最新AI处理意见（shop_evaluation的ai_opinion）
- **商户设备信息**:
  - 显示该商户下所有设备列表，操作:查看设备详情、查看设备维护记录

### 6.3 设备管理
- **设备列表**:
  - 筛选:设备类型、商户
  - 添加设备:表单含商户、类型、标定参数、名称、安装位置、安装人、安装时间、附件、曲线单位、最大/最小值
  - 保存后自动生成二维码（URL为设备详情页）
  - 操作:查看数据（跳曲线图）、编辑、删除
- **设备信息曲线图**:
  - 筛选:设备名称
  - 显示曲线，Y轴刻度自动按最大最小值设置，X轴为时间（最近7天）
- **设备信息维护**:
  - 添加维护记录:设备、状态、施工记录、附件
  - 列表显示所有维护记录

### 6.4 系统管理
- **角色管理**:列表显示角色，点击分配权限弹出菜单树+操作权限树
- **用户管理**:添加/编辑用户，关联角色
- **日志管理**:登录日志列表，支持按时间筛选

### 6.5 任务管理
- **任务列表**:
  - 筛选:状态、责任人（仅管理员可见全部，施工员仅可见自己）
  - 操作:创建、批量删除（仅待处理状态可删除）
  - 列表字段:ID、名称、描述、计划完成时间、状态、责任人、关联商户、更新时间
  - 行内操作:编辑（创建者）、删除（待处理状态）、问题上报（所有人）、完成任务（进行中且是责任人）、验收任务（完成待验收且是监理）
- **任务详情**:显示所有字段
- **施工日志**:
  - 展示所有任务日志（按任务筛选）
  - 添加日志仅通过任务操作触发，不可单独添加
- **日志列表**:全部日志列表，支持按任务筛选

### 6.6 报警管理
- **设备故障管理**:管理故障类型（增删改查）
- **设备报警列表**:
  - 添加报警:设备、故障类型、消息
  - 列表支持编辑、删除

## 七、接口与开发建议

### 7.1 API 返回格式
统一使用 JSON:
```json
{
  "code": 200,
  "msg": "success",
  "data": {}
}
```
- `code`:200成功，4xx客户端错误，5xx服务器错误
- 分页格式:`{ code, msg, data: { list, total, page, limit } }`

### 7.2 认证方式
使用 JWT（Bearer Token），登录接口返回 token，后续请求放入 `Authorization` 头。

### 7.3 文件上传
- 限制单个文件 20MB，支持图片、文档、压缩包
- 附件路径存储相对路径，前端拼接访问

### 7.4 删除校验
- 删除商户前检查是否有设备关联，若有则提示不可删除
- 删除设备前检查是否有未处理报警或进行中任务关联

## 八、初始化数据

系统首次部署时执行 SQL 脚本初始化:
1. 创建数据库及表结构
2. 插入超级管理员用户（`admin` / `admin123`，密码使用 `password_hash` 存储）
3. 插入基础角色:超级管理员、管理员、施工员、工程监理
4. 插入默认菜单（按第五章节结构）
5. 插入权限点（每个菜单对应的增删改查权限）
6. 为超级管理员角色分配所有权限
